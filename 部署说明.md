# Docker 授权部署说明

## 1. 环境要求

| 组件 | 要求 |
|---|---|
| 服务端 | .NET 10（自包含发布，无需安装运行时）、Windows 10+ / Linux |
| 客户端 | GWDataCenter ≥ 6.1.3.0、GWHost ≥ 6.1.2.0 |
| 网络 | 客户端与服务端之间 TCP 互通，单一端口 |

---

## 2. 服务端部署

### 2.1 配置文件

文件：`appsettings.json`（与可执行文件同目录）

```json
{
  "Communication": {
    "DockerBridgeIps": ["172.17.0.1", "172.18.0.1"],
    "TcpPort": 5555,
    "IpAddress": "0.0.0.0"
  }
}
```

| 配置项 | 默认值 | 说明 |
|---|---|---|
| `DockerBridgeIps` | `["172.17.0.1"]` | Linux 下监听的 Docker 网桥地址数组；Windows 下忽略 |
| `TcpPort` | `5000` | TCP 监听端口，Linux/Windows 共用 |
| `IpAddress` | (空) | Windows 下绑定的 IP；为空/`*`/`0.0.0.0` 时监听所有 IP；Linux 下忽略 |

### 2.2 启动运行

#### Linux（systemd 守护进程）

**① 部署文件**：

```bash
mkdir -p /opt/dockerhost
cp -r publish/* /opt/dockerhost/
useradd -r -s /usr/sbin/nologin dockerhost
chown -R dockerhost:dockerhost /opt/dockerhost
```

**② 创建服务** — `/etc/systemd/system/dockerhost.service`：

```ini
[Unit]
Description=Docker Host GUID Service
After=network.target
Wants=network.target

[Service]
Type=notify
ExecStart=/opt/dockerhost/DockerHost
WorkingDirectory=/opt/dockerhost
Restart=always
RestartSec=5
TimeoutStartSec=0

User=dockerhost
Group=dockerhost

Environment=DOTNET_ENVIRONMENT=Production
Environment=ASPNETCORE_ENVIRONMENT=Production

StandardOutput=journal
StandardError=journal
SyslogIdentifier=dockerhost

NoNewPrivileges=true
PrivateTmp=true
ProtectSystem=strict
ProtectHome=true
ReadWritePaths=/opt/dockerhost

[Install]
WantedBy=multi-user.target
```

**③ 启动**：

```bash
systemctl daemon-reload
systemctl enable dockerhost
systemctl start dockerhost
systemctl status dockerhost
```

**④ 查看日志**：

```bash
journalctl -u dockerhost -f
```

#### Windows（Windows Service）

**① 部署文件**：

```powershell
mkdir D:\opt\dockerhost
Copy-Item publish\* D:\opt\dockerhost\
```

**② 创建并启动服务**（管理员 PowerShell）：

```powershell
sc.exe create DockerHostAuth `
    binPath="D:\opt\dockerhost\DockerHost.exe" `
    start=auto `
    obj=LocalSystem

sc.exe start DockerHostAuth
sc.exe query DockerHostAuth
```

**③ 移除服务**：

```powershell
sc.exe stop DockerHostAuth
sc.exe delete DockerHostAuth
```

#### 控制台直接运行（仅调试）

```bash
./DockerHost
```

启动成功后会输出注册码方框和监听地址：

```
[凭证锁] 已获取授权锁: ...
[Windows环境] 正在监听 TCP (所有IP) 端口: 5555
*************************
*                       *
*  软件注册码: XXXX-... *
*                       *
*************************
```

## 3. 客户端部署

### 3.1 配置文件

客户端配置文件位于 `{应用根目录}/data/AlarmCenter/`：

### 3.2 配置说明

文件：`AlarmCenterProperties.xml`

```xml
<?xml version="1.0" encoding="utf-8"?>
<root>
  <Properties name="HostServer">
    <DockerHostIP value="" />
    <DockerHostPort value="5555" />
  </Properties>
</root>
```

| 配置项 | 默认值 | 说明 |
|---|---|---|
| `DockerHostIP` | `host.docker.internal` | 服务端 IP；留空/`*`/`0.0.0.0` 时自动回退到默认值 |
| `DockerHostPort` | `5000` | 服务端端口，必须与服务端 `TcpPort` 一致 |

**不同场景的 `DockerHostIP` 取值**：

| 部署场景 | DockerHostIP | 说明 |
|---|---|---|
| Docker Desktop（Windows/Mac） | 留空或 `host.docker.internal` | 自动解析为宿主机网关 |
| Linux Docker | `host.docker.internal` 或 `172.17.0.1` | `host.docker.internal` 需 `--add-host` 注入 |
| 同机测试（Windows/Linux） | `127.0.0.1` | 不走网卡，不受防火墙影响 |
| 同机测试（指定 IP） | 本机局域网 IP | 需防火墙放行端口 |
| K8s | 宿主机 IP（通过 `status.hostIP` 注入） | 服务端使用 `hostPort` 映射 |

### 3.3 容器启动

#### Docker Desktop（Windows/Mac）

`host.docker.internal` 自动可用，无需额外配置：

```bash
docker run -d your-client-image
```

#### Linux Docker

需要注入 `host-gateway`：

```bash
docker run -d \
  --add-host host.docker.internal:host-gateway \
  your-client-image
```

#### 指定网桥 IP（备选）

```bash
docker run -d your-client-image
# 同时在 AlarmCenterProperties.xml 中设置:
# <DockerHostIP value="172.17.0.1" />
```

### 3.4 故障排查

| 现象 | 可能原因 | 解决 |
|---|---|---|
| `获取 Docker 授权码超时` | 服务端未启动 | 先启动服务端 |
| | 端口不一致 | 检查客户端 `DockerHostPort` 与服务端 `TcpPort` |
| | 配置文件未读到 | 检查 `AlarmCenterProperties.xml` 路径和格式 |
| | 系统代理拦截 | 确认无需额外配置（客户端已默认禁用代理） |
| | Windows 防火墙拦截 | 放行服务端监听端口 |
| `连接丢失 → 重连` 循环 | 客户端心跳间隔过长 | 无需配置（客户端已默认设为 3 秒心跳） |
| `拒绝重复连接` | 同一 GUID 已有活跃连接 | 停止旧实例或更换 GUID |

## 4. Kubernetes 部署

### 4.1 网络架构

```
+-------------------------------------------------------+
| K8s Node                                                |
|                                                         |
|  +---------------------------------------------------+ |
|  | DaemonSet: dockerhost-server                       | |
|  |  hostPort: 5555 (映射到宿主机)                       | |
|  |  privileged: true (读取宿主机硬件信息)                 | |
|  |  凭证锁 /var/tmp/gw-auth-{MD5}.lock                 | |
|  +-----------------------+-----------------------------+ |
|                          |                                |
|                          | TCP :5555 (宿主机端口)           |
|                          v                                |
|  +-----------------------+-----------------------------+ |
|  | Deployment: client-app                              | |
|  |  env: DOCKER_HOST_IP = status.hostIP (Downward API) | |
|  |  连接 $(DOCKER_HOST_IP):5555                         | |
|  +---------------------------------------------------+ |
+-------------------------------------------------------+
```

### 4.2 设计约束

| 约束 | 方案 |
|---|---|
| 客户端必须与服务端同节点 | DaemonSet 每节点一个服务端，客户端通过 `podAffinity` 对齐 |
| 外部不可访问授权端口 | `hostPort` 仅绑定宿主机指定接口，配合 NetworkPolicy 限制 |
| Pod 安全策略合规 | 优先 `hostPort`（仅占单端口），非 `hostNetwork`（暴露完整网络栈） |
| 服务端读取宿主机硬件 | `privileged: true`，或 `SYS_PTRACE` + hostPath 挂载 |
| 凭证锁防同节点多开 | `/var/tmp` 通过 hostPath 挂载宿主机目录 |
| **服务端监听地址（关键）** | `DockerBridgeIps` 必须设为 `["0.0.0.0"]`：K8s Pod 无 Docker 网桥，默认 `172.17.0.1` 会绑定失败回退到 `127.0.0.1`，导致 hostPort 流量无法到达服务端 |

### 4.3 服务端 DaemonSet

> **前提**：命名空间需允许 `privileged` Pod。见 [4.7 Pod Security Standards](#47-pod-security-standards)。

```yaml
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: dockerhost-server
  namespace: gw-auth
  labels:
    app: dockerhost-server
spec:
  selector:
    matchLabels:
      app: dockerhost-server
  template:
    metadata:
      labels:
        app: dockerhost-server
    spec:
      hostPID: false
      containers:
      - name: server
        image: your-registry/dockerhost:latest
        imagePullPolicy: IfNotPresent
        ports:
        - name: auth
          containerPort: 5555
          hostPort: 5555
          protocol: TCP
        securityContext:
          privileged: true
        env:
        - name: ASPNETCORE_ENVIRONMENT
          value: "Production"
        - name: Communication__TcpPort
          value: "5555"
        # 关键：K8s 下必须监听 0.0.0.0，否则 hostPort 流量无法到达
        - name: Communication__DockerBridgeIps__0
          value: "0.0.0.0"
        volumeMounts:
        - name: vartmp
          mountPath: /var/tmp
        - name: machine-id
          mountPath: /etc/machine-id
          readOnly: true
        - name: cpuinfo
          mountPath: /proc/cpuinfo
          readOnly: true
        livenessProbe:
          tcpSocket:
            port: 5555
          initialDelaySeconds: 10
          periodSeconds: 15
        readinessProbe:
          tcpSocket:
            port: 5555
          initialDelaySeconds: 5
          periodSeconds: 10
        resources:
          requests:
            cpu: 50m
            memory: 64Mi
          limits:
            cpu: 200m
            memory: 128Mi
      volumes:
      - name: vartmp
        hostPath:
          path: /var/tmp
          type: DirectoryOrCreate
      - name: machine-id
        hostPath:
          path: /etc/machine-id
          type: File
      - name: cpuinfo
        hostPath:
          path: /proc/cpuinfo
          type: File
```

### 4.4 客户端 Deployment

> **说明**：客户端代码从 `AlarmCenterProperties.xml` 读取 `DockerHostIP`，不读环境变量。因此通过 **initContainer** 在启动前将 `status.hostIP` 注入 XML，然后复制到 emptyDir 供主容器挂载。

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: client-app
  namespace: gw-auth
  labels:
    app: client-app
spec:
  replicas: 1
  selector:
    matchLabels:
      app: client-app
  template:
    metadata:
      labels:
        app: client-app
    spec:
      initContainers:
      - name: inject-docker-host-ip
        image: busybox:stable
        command:
        - sh
        - -c
        - |
          cp /config-in/AlarmCenterProperties.xml /config-out/AlarmCenterProperties.xml
          if [ -n "${DOCKER_HOST_IP}" ]; then
            sed -i "s|<DockerHostIP value=\"\" />|<DockerHostIP value=\"${DOCKER_HOST_IP}\" />|" /config-out/AlarmCenterProperties.xml
            echo "DockerHostIP 已注入: ${DOCKER_HOST_IP}"
          fi
        env:
        - name: DOCKER_HOST_IP
          valueFrom:
            fieldRef:
              fieldPath: status.hostIP
        volumeMounts:
        - name: license-config-template
          mountPath: /config-in
        - name: license-config
          mountPath: /config-out
      containers:
      - name: client
        image: your-registry/gwdatacenter:latest
        imagePullPolicy: IfNotPresent
        env:
        - name: DOCKER_HOST_PORT
          value: "5555"
        volumeMounts:
        - name: license-config
          mountPath: /app/data/AlarmCenter
          readOnly: true
        resources:
          requests:
            cpu: 100m
            memory: 256Mi
          limits:
            cpu: "2"
            memory: 2Gi
      volumes:
      - name: license-config-template
        configMap:
          name: client-license-config
      - name: license-config
        emptyDir: {}
      affinity:
        podAffinity:
          requiredDuringSchedulingIgnoredDuringExecution:
          - labelSelector:
              matchLabels:
                app: dockerhost-server
            topologyKey: kubernetes.io/hostname
```

### 4.5 客户端 ConfigMap

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: client-license-config
  namespace: gw-auth
data:
  AlarmCenterProperties.xml: |
    <?xml version="1.0" encoding="utf-8"?>
    <root>
      <Properties name="HostServer">
        <DockerHostIP value="" />
        <DockerHostPort value="5555" />
      </Properties>
    </root>
```

> **IP 注入原理**：`AlarmCenterProperties.xml` 中 `DockerHostIP` 留空占位。4.4 的 initContainer 在 Pod 启动前用 `status.hostIP` 替换空值，生成最终的配置文件到 emptyDir。主容器挂载 emptyDir，读到的是已注入宿主机 IP 的有效配置。

### 4.6 NetworkPolicy（可选，有局限性）

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: dockerhost-auth-policy
  namespace: gw-auth
spec:
  podSelector:
    matchLabels:
      app: dockerhost-server
  policyTypes:
  - Ingress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: client-app
    ports:
    - protocol: TCP
      port: 5555
```

> **局限性**：`hostPort` 流量经宿主机 iptables DNAT 进入 Pod，部分 CNI（Calico、Cilium 等）的 NetworkPolicy 实现**可能不拦截 hostPort 路径**。此 NetworkPolicy 能限制 Pod 网络层的直接访问，但不保证覆盖 hostPort 映射流量。若需严格的端口访问控制，建议额外使用节点级防火墙（如 `iptables` 规则限制 `hostPort` 仅允许本节点 Pod CIDR）。

### 4.7 Pod Security Standards

服务端 DaemonSet 使用 `privileged: true`，K8s 1.25+ 的 Pod Security Standards (PSS) 默认拒绝此类 Pod。部署前需给命名空间打标签：

```bash
kubectl label ns gw-auth pod-security.kubernetes.io/enforce=privileged
kubectl label ns gw-auth pod-security.kubernetes.io/audit=privileged
kubectl label ns gw-auth pod-security.kubernetes.io/warn=privileged
```

| 标签 | 作用 |
|---|---|
| `enforce=privileged` | 允许 privileged Pod 运行，拒绝不满足的策略 |
| `audit=privileged` | 仅审计记录，不拒绝（排查阶段使用） |
| `warn=privileged` | 向用户发送警告，不拒绝 |

> **注意**：若集群使用 Pod Security Admission (PSA) 以外的准入控制器（如 OPA/Gatekeeper、Kyverno），需按其语法编写对应的例外策略。

## 5. 端口与防火墙

| 方向 | 端口 | 协议 | 说明 |
|---|---|---|---|
| 客户端 → 服务端 | 5555（可配置） | TCP | SignalR WebSocket 长连接 |
| 服务端 Kestrel | 5555（可配置） | TCP | 同机测试可用 `127.0.0.1` 绕过防火墙 |

**Windows 防火墙放行**：

```powershell
netsh advfirewall firewall add rule name="DockerHostAuth" dir=in action=allow protocol=tcp localport=5555
```
